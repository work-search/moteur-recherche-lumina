Introduction

L'API Work Search permet de rechercher des articles dans la base de données en fonction d'un terme de recherche fourni. Elle renvoie les résultats au format JSON.

Endpoint

/api/article/<recherche>

Méthode

GET

Paramètres

  • recherche (obligatoire) : Le terme de recherche à utiliser pour trouver des articles.

Réponse

La réponse est au format JSON et contient les éléments suivants :

  • articles : Un tableau d'objets représentant les articles trouvés. Chaque objet article contient :
    • id : L'identifiant unique de l'article
    • title : Le titre de l'article
    • description : La description de l'article
    • url : L'URL de l'article

Exemple de réponse

{
  "articles": [
    {
      "id": 1,
      "title": "Exemple d'article",
      "description": "Ceci est une description d'exemple",
      "url": "https://example.com/article1"
    },
    {
      "id": 2,
      "title": "Autre article",
      "description": "Une autre description",
      "url": "https://example.com/article2"
    }
  ]
}

Codes d'erreur

  • 500 : Erreur interne du serveur (par exemple, problème de connexion à la base de données)

Limites

  • La recherche est limitée à 50 résultats maximum.
  • La recherche s'effectue sur les champs "title" et "description" des articles.

Exemple d'utilisation

GET /api/article/python

Cette requête recherchera tous les articles contenant le terme "python" dans leur titre ou leur description.
